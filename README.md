# ml-signal-backtester

This project uses machine learning to build a trading strategy and compare its returns against a simple buy-and-hold approach. It pulls data for 5 tickers and engineers features from raw price data in order to classify days as gains or losses. Specifically, we use RSI (to identify whether a stock is overbought or oversold by measuring the momentum of its price changes), Volatility (to measure how erratic a stock's movement is, calculated as the standard deviation of returns over a 20-day rolling window), MA Spread (how far the current price sits from its 20-day moving average, capturing medium-term trend direction), and multi-day returns (broken into 5-day and 20-day lookback windows to capture short and medium-term momentum).
The data is split 80/20 chronologically — not randomly — because random shuffling would leak future data into training, making the model appear more accurate than it actually is. The signal is converted to +1 (long) or -1 (short) based on the model's prediction, and shifted forward by one day before multiplying against actual returns to prevent lookahead bias — you can only act on a signal the day after you receive it.
The last 20% of data is used to evaluate the model using F1-macro and MCC. MCC scores near zero across all five tickers indicate the model has no real predictive skill. This is reflected in the backtest results: AMZN returned +42% while LLY lost -47%. With near-zero predictive signal, outcomes scatter randomly — some strategies look lucky, others look terrible, purely by chance rather than genuine edge.
